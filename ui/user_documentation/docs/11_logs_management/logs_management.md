# 11. Logs Management

Centralized viewer for operational logs produced by the platform. Two log sources are available, selectable via a radio button at the top of the page.

---

## Log Sources

| Source | Description |
|---|---|
| **Deletion Logs** | Timestamped records of entity deletions performed through the Admin Console |
| **Datatrove Pipeline Logs** | Per-task `.log` files generated by Datatrove pipeline execution sessions |

---

## Deletion Logs

Deletion logs are written to `ui/admin_console/logs/` whenever a dataset, distribution, or dataset card is deleted via the Admin Console.

The view lists all available `.log` files sorted by modification time (most recent first). Click any file name to display its content below the list.

Each log file records:
- The deletion timestamp
- Entity type, UUID, and name for every deleted entity (including cascade-deleted children)

---

## Datatrove Pipeline Logs

Datatrove logs are organised in **session folders** under `logs/` at the project root. Each pipeline run creates one session folder containing a `logs/` subdirectory with one `.log` file per Datatrove task.

### Browsing Sessions

A dropdown lists all session folders sorted by name (most recent first). The total session count is shown above the dropdown.

### Filtering Log Files

Within the selected session, filters can be applied before rendering the file list:

| Filter | Description |
|---|---|
| **Sort by lines** | Ascending or Descending order by line count |
| **Min lines** | Exclude files with fewer than N lines |
| **Max lines** | Exclude files with more than N lines (0 = no upper limit) |
| **Search keyword** | Show only files whose content contains the given string (case-insensitive) |

The file count matching the active filters is shown above the list.

### Viewing a Log File

Click any file button to load its content into a syntax-highlighted code block (`language="log"`) below the list.

---

## Deleting Logs

### Delete a Single Session

At the bottom of the Datatrove Logs section, click **🗑️ Delete session folder** to permanently remove the currently selected session directory (including all its log files). A confirmation step is required before deletion proceeds.

### Delete All Sessions

Click **🗑️ Delete ALL sessions** to remove every session folder under `logs/`. A two-step confirmation is required. This action is irreversible and frees all disk space used by pipeline logs.

> **Note:** Deletion log files (from the Admin Console) are not affected by these actions.
